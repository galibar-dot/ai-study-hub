## API 配置管理

### ✅ 功能概述

通过前端界面管理 API 提供商和模型，无需手动编辑配置文件。

### 🚀 使用方法

#### 1. 打开设置

1. 点击侧边栏底部的 "⚙️ 设置" 按钮
2. 点击 "🔧 API 配置"

#### 2. 添加 API 提供商

**步骤**:
1. 点击 "➕ 添加 API 提供商"
2. 填写表单：
   ```
   提供商名称: openai-gpt4o
   API 地址: https://api.openai.com/v1
   API 密钥: sk-proj-xxxxx
   模型名称: gpt-4o
   接口类型: Responses API (OpenAI 新版)
   ```
3. 点击 "测试连接" 验证（可选）
4. 点击 "保存"

#### 3. 切换模型

1. 点击输入框上方的模型按钮（例如 "Opus 4.7"）
2. 在弹出的模型列表中选择：
   - Claude 模型（claude-opus-4-7, claude-sonnet-4-6, claude-haiku-4-5）
   - 或你添加的 API 模型（gpt-4o, gpt-4o-mini 等）
3. 选中后立即生效

#### 4. 编辑/删除 API 提供商

1. 打开设置页面
2. 点击提供商卡片上的 "✏️" 编辑按钮
3. 修改配置后点击 "保存"
4. 或点击 "删除提供商" 按钮删除

### 🎯 设计特点

- **简化设计**：一次只填一个模型，避免复杂输入
- **安全性**：API 密钥在列表中脱敏显示（`sk-••••••••••••••••••••`）
- **测试连接**：保存前验证 API 是否可用
- **实时反馈**：显示成功/失败消息
- **自动刷新**：保存后立即更新模型列表

### 🔧 技术实现

**后端**:
- 使用 `yaml` 库读写 `config.yml`
- 保存后重新加载配置并重建 `relay_clients`
- 支持热更新，无需重启服务

**前端**:
- 动态加载模型列表（不再硬编码）
- 分组显示 Claude 模型和 API 模型
- 使用 Sheet 弹窗实现设置界面

## 多模型支持

### ✅ 新增功能

#### 1. 一个提供商支持多个模型

现在可以为同一个 API 提供商添加多个模型，共享同一个 API 密钥和地址。

#### 2. 两种 API 接口类型

支持两种主流的 API 接口格式：

| 接口类型 | 适用于 | 端点 |
|---------|-------|------|
| **Responses API** | OpenAI GPT (新版) | `/v1/responses` |
| **Chat Completions API** | DeepSeek、Claude、通用 API | `/v1/chat/completions` |

#### 3. 灵活的模型管理

- **添加模型**：点击提供商卡片上的 **➕** 按钮
- **删除模型**：点击模型标签上的 **×** 按钮
- **删除提供商**：删除所有模型

### 🚀 使用示例

#### 示例 1：添加 DeepSeek 的多个模型

**第一步：创建提供商并添加第一个模型**
```
提供商名称: deepseek
API 地址: https://api.deepseek.com/v1
API 密钥: sk-xxxxx
模型名称: deepseek-chat
接口类型: Chat Completions API (DeepSeek / 通用)
```
点击 "保存"

**第二步：添加第二个模型**
1. 在设置页面找到 "deepseek" 提供商
2. 点击 **➕** 按钮
3. 填写：
   ```
   模型名称: deepseek-coder
   API 密钥: (留空，使用原有密钥)
   ```
4. 点击 "保存"

**结果**：现在 "deepseek" 提供商下有两个模型：
- `deepseek-chat`
- `deepseek-coder`

两个模型共享同一个 API 密钥和地址。

#### 示例 2：添加 OpenAI 的多个模型

**第一步：创建提供商**
```
提供商名称: openai
API 地址: https://api.openai.com/v1
API 密钥: sk-proj-xxxxx
模型名称: gpt-4o
接口类型: Responses API (OpenAI 新版 / GPT)
```

**第二步：继续添加其他模型**

点击 **➕** 按钮，依次添加：
- `gpt-4o-mini`
- `o1`
- `o1-mini`

**结果**："openai" 提供商下有 4 个模型，都使用同一个 API 密钥。

### 📋 界面说明

#### 提供商卡片显示
```
┌─────────────────────────────────────┐
│ deepseek                        ➕ 🗑│
│ https://api.deepseek.com/v1         │
│ [deepseek-chat ×] [deepseek-coder ×]│
│ | Chat Completions                  │
└─────────────────────────────────────┘
```

- **提供商名称**：deepseek
- **API 地址**：https://api.deepseek.com/v1
- **模型标签**：每个模型都有 × 按钮可以单独删除
- **接口类型**：Chat Completions
- **➕ 按钮**：添加新模型到此提供商
- **🗑 按钮**：删除整个提供商及所有模型

### 🔧 配置文件格式

```yaml
relay:
  default_provider: "deepseek"
  providers:
    deepseek:
      base_url: "https://api.deepseek.com/v1"
      api_key: "sk-xxxxx"
      api_type: "chat_completions"
      models:
        - "deepseek-chat"
        - "deepseek-coder"
    
    openai:
      base_url: "https://api.openai.com/v1"
      api_key: "sk-proj-xxxxx"
      api_type: "responses"
      models:
        - "gpt-4o"
        - "gpt-4o-mini"
        - "o1"
```

### 💡 使用建议

1. **按提供商分组**：同一个 API 提供商的模型放在一起
2. **命名规范**：提供商名称用小写，如 `deepseek`、`openai`
3. **测试连接**：添加第一个模型时测试连接，确保配置正确
4. **模型命名**：使用官方的模型名称，如 `deepseek-chat`、`gpt-4o`

### 🎯 常见场景

**场景 1：DeepSeek 多模型**
```
deepseek
├── deepseek-chat (对话)
└── deepseek-coder (编程)
```

**场景 2：OpenAI 全家桶**
```
openai
├── gpt-4o (最强)
├── gpt-4o-mini (快速)
├── o1 (推理)
└── o1-mini (推理快速)
```

**场景 3：多个中转服务**
```
relay1
├── gpt-4o
└── claude-3-opus

relay2
├── gpt-4o-mini
└── claude-3-sonnet
```

### ⚠️ 注意事项

1. **删除模型**：删除最后一个模型时，整个提供商也会被删除
2. **API 密钥**：添加模型时留空 API 密钥，会使用原有密钥
3. **接口类型**：同一个提供商的所有模型使用相同的接口类型
4. **模型去重**：不能添加重复的模型名称

## 英语学习功能

### ✅ 功能概述

完整的英语学习系统，包括：
- 点击单词即时翻译
- 生词本管理
- 学习统计

### 📥 安装步骤

#### Step 1: 下载词典数据库

**方法 1：手动下载（推荐）**

1. 访问 ECDICT 项目的 Release 页面：
   ```
   https://github.com/skywind3000/ECDICT/releases
   ```

2. 下载最新版本的 SQLite 数据库文件：
   - 文件名：`ecdict-sqlite-28.zip`（或更新版本）
   - 大小：约 50MB

3. 解压缩后，将 `stardict.db` 文件重命名为 `ecdict.db`

4. 将 `ecdict.db` 文件放到项目根目录：
   ```
   e:\python_project\claude code1\ecdict.db
   ```

**方法 2：使用镜像**

如果 GitHub 访问困难，可以使用以下镜像地址：
```
https://ghproxy.com/https://github.com/skywind3000/ECDICT/releases/download/1.0.28/ecdict-sqlite-28.zip
```

#### Step 2: 测试后端功能（可选）

```powershell
cd "e:\python_project\claude code1"
python test_learning.py
```

如果看到 "🎉 所有测试通过！" 说明配置正确。

#### Step 3: 启动服务器

```powershell
cd "e:\python_project\claude code1"
python server.py
```

### 📖 使用说明

#### 1. 切换到学习模式

- 点击顶部的 "📖 学习" 按钮
- 学习模式下可以双击单词查询翻译

#### 2. 查询单词

- 在聊天内容中双击任意英文单词
- 会弹出翻译卡片，显示：
  - 音标
  - 中文释义
  - 英文释义
  - 词性
  - 标签（牛津3000词、柯林斯星级）

#### 3. 添加到生词本

- 在翻译卡片中点击 "➕ 加入生词本" 按钮
- 单词会保存到数据库中

#### 4. 管理生词本

- 点击侧边栏的 "设置" → "📚 生词本"
- 可以查看所有生词
- 查看学习统计（总词数、今日新增、复习次数）
- 删除单词：点击单词右侧的 × 按钮
- 导出生词本：点击 "导出" 按钮，生成 CSV 文件
- 清空生词本：点击 "清空" 按钮

### 🎯 功能特点

#### 核心优势
- ✅ **完全离线**：无需联网，查询速度快
- ✅ **无限制**：不消耗 API 额度，不浪费 token
- ✅ **数据丰富**：77 万词条，包含音标、释义、词频等
- ✅ **跨设备同步**：生词本存储在服务器端，可跨设备访问

#### 数据质量
- 牛津3000词标注
- 柯林斯星级标注
- 英国国家语料库词频
- 完整的词形变化

### 💡 使用技巧

#### 1. 高效查词
- 在 AI 回复的英文内容中双击单词
- 快速了解生词含义
- 一键添加到生词本

#### 2. 生词复习
- 定期打开生词本复习
- 导出 CSV 文件到其他工具
- 利用统计数据了解学习进度

#### 3. 学习建议
- 每天查询 10-20 个新单词
- 定期复习生词本
- 结合 AI 对话练习使用

### 🔧 故障排查

#### 问题 1：启动时报错 "词典数据库不存在"
**原因**：未下载或未正确放置 `ecdict.db` 文件

**解决**：
1. 检查文件是否存在：`e:\python_project\claude code1\ecdict.db`
2. 确认文件名是 `ecdict.db`（不是 `stardict.db`）
3. 确认文件大小约 50MB

#### 问题 2：双击单词没有反应
**原因**：未切换到学习模式

**解决**：
1. 点击顶部的 "📖 学习" 按钮
2. 确认按钮显示为高亮状态

#### 问题 3：查询单词显示 "未找到该单词"
**原因**：
- 单词拼写错误
- 单词不在词典中（生僻词）
- 选中的不是单个单词

**解决**：
1. 检查拼写
2. 确保双击选中的是单个完整单词
3. 词典包含常用词汇，生僻词可能不在其中

#### 问题 4：生词本无法保存
**原因**：数据库权限问题

**解决**：
1. 检查 `chats` 目录是否存在
2. 确认有写入权限
3. 查看服务器日志是否有错误信息

### 📊 数据库信息

**词典数据库 (ecdict.db)**
- 位置：项目根目录
- 大小：约 50MB
- 词条数：77 万+
- 表名：`stardict`

**生词本数据库 (vocabulary.db)**
- 位置：`chats/vocabulary.db`
- 自动创建
- 表名：`vocabulary`

### 📝 注意事项

1. **词典数据库是只读的**，不会被修改
2. **生词本数据存储在服务器端**，清除浏览器数据不会丢失
3. **学习模式状态保存在浏览器**，刷新页面会保持
4. **移动端暂不支持模式切换器**（空间限制），但功能正常

## 阅读专项功能

### ✅ 功能概述

完整的英语阅读理解练习系统，包括：
- 文章库管理
- 阅读理解题目
- 答题记录和统计
- 难度分级

### 🚀 使用方法

#### 1. 进入阅读专项

1. 点击侧边栏的 "设置" 按钮
2. 点击 "📖 阅读专项"

#### 2. 选择文章

1. 使用难度筛选（全部/初级/中级/高级）
2. 点击文章卡片开始阅读

#### 3. 阅读和答题

1. 阅读文章内容（可滚动）
2. 回答题目（固定在底部）
3. 使用 "上一题" / "下一题" 导航
4. 双击单词可查词（无需切换模式）
5. 完成后点击 "提交答案"

#### 4. 查看结果

1. 查看得分和正确率
2. 查看每道题的对错情况
3. 阅读错题解析
4. 选择 "返回列表" 或 "再做一次"

### 📊 功能特点

#### 文章管理
- 支持三个难度级别（初级/中级/高级）
- 文章分类（科技、环境、教育等）
- 自动统计字数
- 显示题目数量

#### 答题系统
- 单选题支持
- 实时保存答案
- 题目导航（上一题/下一题）
- 提交前检查未答题目

#### 结果分析
- 实时计算得分
- 显示正确/错误状态
- 提供详细解析
- 支持重做练习

#### 学习统计
- 完成文章数
- 总题目数和正确率
- 平均分
- 各难度完成情况

### 🎨 UI 设计

#### 颜色标识
- 🟢 初级 - 绿色（#4ade80）
- 🟡 中级 - 黄色（#fbbf24）
- 🔴 高级 - 红色（#f87171）

#### 交互设计
- 文章卡片悬停效果
- 选项选中高亮
- 按钮禁用状态
- 流畅的页面切换

### 🎓 学习建议

#### 初学者
1. 从初级文章开始
2. 先完整阅读文章
3. 遇到生词双击查询
4. 答题后仔细阅读解析

#### 进阶学习
1. 挑战中级和高级文章
2. 限时阅读提高速度
3. 记录错题重点复习

### 💡 使用技巧

#### 高效学习
1. **先阅读后答题** - 完整阅读文章再答题
2. **结合生词本** - 双击生词添加到生词本
3. **重复练习** - 错题文章可以重做
4. **循序渐进** - 从初级到高级逐步提升

#### 答题技巧
1. 仔细阅读题目
2. 排除明显错误选项
3. 回到文章找依据
4. 注意关键词定位
