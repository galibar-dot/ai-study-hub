### Phase 4: 多模型支持增强
**完成时间**: 第四阶段

#### ✅ 1. 一个提供商支持多个模型

现在可以为同一个 API 提供商添加多个模型，共享同一个 API 密钥和地址。

#### ✅ 2. 两种 API 接口类型

| 接口类型 | 适用于 | 端点 |
|---------|-------|------|
| **Responses API** | OpenAI GPT (新版) | `/v1/responses` |
| **Chat Completions API** | DeepSeek、Claude、通用 API | `/v1/chat/completions` |

#### ✅ 3. 灵活的模型管理

- 添加模型：点击提供商卡片上的 **➕** 按钮
- 删除模型：点击模型标签上的 **×** 按钮
- 删除提供商：删除所有模型

#### 技术实现

**后端改动**:
1. `RelayProvider` 模型：添加 `api_type` 字段
2. 保存接口：检测提供商是否已存在，添加新模型到模型列表
3. 删除模型接口：`DELETE /api/config/providers/{name}/models/{model}`
4. API 调用：根据 `api_type` 选择不同的接口格式

**配置文件格式**:
```yaml
relay:
  default_provider: "deepseek"
  providers:
    deepseek:
      base_url: "https://api.deepseek.com/v1"
      api_key: "sk-xxxxx"
      api_type: "chat_completions"
      models:
        - "deepseek-chat"
        - "deepseek-coder"
```

**功能入口**:
- 阅读专项页面顶部的 "✨ AI 生成新文章" 按钮

**生成流程**:
