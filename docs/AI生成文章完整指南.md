## 前端生成功能

### ✨ 功能入口

**位置：** 设置 → 阅读专项 → "✨ AI 生成新文章" 按钮

### 🚀 使用流程

1. 点击"AI 生成新文章"按钮
2. 选择 AI 模型（支持 Claude 和自定义 API 模型）
3. 输入文章主题（建议用英文，如 "Climate Change"）
4. 选择生成数量（1-3 篇）
5. 点击"开始生成"
6. 实时显示生成进度
7. 生成完成后自动保存到数据库并刷新列表

### 🎨 界面特点

- **流式生成**：使用 SSE 实时显示生成进度
- **双模型支持**：同时支持 Claude SDK 和 OpenAI API
- **智能解析**：自动提取 JSON 内容（支持 ```json 包裹）
- **响应式设计**：手机端和桌面端都有优化

### 📊 生成的文章格式

**文章信息：**
- 标题（title）
- 内容（content，400-500 词）
- 难度（difficulty: beginner/intermediate/advanced）

**题目信息（5 道）：**
1. 主旨题（Main idea）
2. 细节题（Specific detail）
3. 推理题（Inference）
4. 词汇题（Vocabulary in context）
5. 态度题（Attitude/Tone）

**每道题包含：**
- 题目文本（question）
- 4 个选项（options: A/B/C/D）
- 正确答案（answer）
- 解析（explanation）

## 后端脚本生成

### 🎯 系统特点

- ✅ **精心优化的提示词** - 专门针对六级考试标准
- ✅ **多模型支持** - 可选择 GPT-5.5、DeepSeek 等模型
- ✅ **批量生成** - 一次生成多篇文章
- ✅ **自动保存** - 直接添加到数据库
- ✅ **质量保证** - 严格的字数、题目数量控制

### 📝 提示词特点

#### 文章要求
- **长度**: 严格控制在 400-500 词
- **结构**: 引言 → 论证 → 结论
- **词汇**: 六级核心词汇
- **句式**: 复杂句、从句、被动语态
- **内容**: 包含研究、数据、专家观点

#### 题目设计
1. **题目 1**: 主旨大意 / 作者意图
2. **题目 2**: 细节理解
3. **题目 3**: 推理判断
4. **题目 4**: 词汇理解 / 指代关系
5. **题目 5**: 作者态度 / 建议

#### 干扰项设计
- 部分正确但不完整
- 真实陈述但不回答问题
- 与正确答案相反
- 混合不同段落的信息

### 🎨 主题库

系统内置 30 个主题，涵盖 6 大类别：

**教育类 (Education)**
- 在线学习效果、学生心理健康、教育不平等、标准化考试争议、终身学习重要性

**科技类 (Technology)**
- 人工智能伦理、数字时代隐私、社交媒体影响、自动化与就业、网络安全挑战

**环境类 (Environment)**
- 气候变化解决方案、可再生能源转型、塑料污染、城市可持续发展、生物多样性保护

**社会类 (Society)**
- 工作生活平衡、收入不平等、人口老龄化、文化多样性、城市化趋势

**健康类 (Health)**
- 心理健康意识、预防性医疗、营养与饮食、运动益处、睡眠重要性

**经济类 (Economy)**
- 零工经济增长、加密货币争议、消费者行为变化、全球贸易紧张、可持续商业实践

### 💡 使用建议

#### 首次使用

## API 接口说明

### 🔧 后端 API 端点

#### 1. 生成文章
```http
POST /api/reading/generate
Content-Type: application/json

{
  "model": "claude-opus-4-7",
  "topic": "Climate Change",
  "count": 1
}

Response: Server-Sent Events (SSE)
data: {"type": "text", "text": "..."}
data: {"type": "done", "articles": [{"id": 1, "title": "..."}]}
data: {"type": "error", "error": "..."}
```

#### 2. 删除文章
```http
DELETE /api/reading/articles/{article_id}

Response:
{
  "ok": true
}
```

#### 3. 更新文章（预留接口）
```http
PUT /api/reading/articles/{article_id}
Content-Type: application/json

{
  "title": "新标题",
  "content": "新内容",
  "difficulty": "intermediate"
}

Response:
{
  "ok": true
}
```

- 删除前有确认提示，防止误操作

## 故障排查

### 🐛 常见问题

#### 问题 1: JSON 解析失败
**原因**: AI 返回的不是纯 JSON  
**解决**: 
- 系统会自动移除 markdown 标记
- 如果仍失败，检查 API 响应

#### 问题 2: 生成的文章太短/太长
**原因**: 模型没有严格遵守字数要求  
**解决**:
- 使用 gpt-5.5 或 deepseek-v4-pro
- 在提示词中强调字数要求

#### 问题 3: 题目质量不高
**原因**: 干扰项设计不够巧妙  
**解决**:
- 手动调整题目
- 或重新生成该文章

#### 问题 4: API 调用失败
**原因**: 网络问题或 API 配置错误  
**解决**:
- 检查网络连接
- 验证 API 密钥是否正确
- 确认 API 地址格式

#### 问题 5: 生成失败（前端）
**解决步骤**:
- 检查模型是否配置正确
- 检查 API 密钥是否有效
- 检查网络连接
- 查看浏览器控制台错误

#### 问题 6: 删除失败
**解决步骤**:
- 检查文章是否存在
- 检查是否有权限
- 刷新页面重试

#### 问题 7: 列表不更新
**解决步骤**:
- 手动刷新页面（Ctrl+F5）
- 检查浏览器控制台错误

## 🎓 质量标准

### 优秀文章标准
- ✅ 字数在 400-500 之间
- ✅ 结构清晰，逻辑连贯
- ✅ 使用六级词汇和句式
- ✅ 包含具体例子或数据
- ✅ 5 道题目覆盖不同题型
- ✅ 干扰项设计巧妙
- ✅ 答案解析清晰准确

### 需要改进的标志
- ❌ 字数偏差超过 50 词
- ❌ 词汇过于简单或复杂
- ❌ 题目答案过于明显
- ❌ 干扰项不合理
- ❌ 解析不够详细
