# 手动推送命令（复制粘贴版）

如果自动脚本有问题，请按照以下步骤手动执行：

---

## 第一步：配置 Git（只需一次）

打开 PowerShell 或命令提示符，执行：

```cmd
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

**示例**：
```cmd
git config --global user.name "Zhang San"
git config --global user.email "zhangsan@example.com"
```

---

## 第二步：进入项目目录并提交

```cmd
cd /d "E:\python_project\claude code1"
git commit -m "Initial commit: AI Study Hub v1.0.0"
```

---

## 第三步：创建远程仓库

### GitHub
1. 访问：https://github.com/new
2. 仓库名：`ai-study-hub`
3. 不要勾选 "Add a README file"
4. 点击 "Create repository"

### Gitee
1. 访问：https://gitee.com/projects/new
2. 仓库名：`ai-study-hub`
3. 不要勾选 "使用 Readme 文件初始化"
4. 点击 "创建"

---

## 第四步：推送代码

### 推送到 GitHub

```cmd
git remote add origin https://github.com/你的用户名/ai-study-hub.git
git branch -M main
git push -u origin main
```

**完整示例**（假设用户名是 zhangsan）：
```cmd
git remote add origin https://github.com/zhangsan/ai-study-hub.git
git branch -M main
git push -u origin main
```

推送时输入：
- Username: 你的 GitHub 用户名
- Password: Personal Access Token（获取地址：https://github.com/settings/tokens）

---

### 推送到 Gitee

```cmd
git remote add origin https://gitee.com/你的用户名/ai-study-hub.git
git push -u origin master
```

**完整示例**（假设用户名是 zhangsan）：
```cmd
git remote add origin https://gitee.com/zhangsan/ai-study-hub.git
git push -u origin master
```

推送时输入：
- Username: 你的 Gitee 用户名
- Password: 你的 Gitee 密码

---

## 常见错误处理

### 错误：remote origin already exists
说明已经关联过了，先删除再重新关联：
```cmd
git remote remove origin
git remote add origin https://github.com/你的用户名/ai-study-hub.git
```

### 错误：Author identity unknown
需要先配置 Git 用户信息（见第一步）

### 错误：Repository not found
1. 确认仓库已创建
2. 确认用户名正确
3. 确认仓库名是 `ai-study-hub`
