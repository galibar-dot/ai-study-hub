## 报告安全漏洞

如果你发现了安全漏洞，请**不要**公开提交 Issue。

请通过以下方式私下报告：

1. 发送邮件到项目维护者（如果有）
2. 或在 GitHub 上使用 Security Advisory 功能

我们会尽快响应并处理安全问题。

## 安全最佳实践

### 部署建议

1. **修改默认密码**
   - 在 `server.py` 中修改 `PASSWORD` 变量
   - 使用强密码（至少 12 位，包含大小写字母、数字、特殊字符）

2. **保护敏感文件**
   - 不要将 `.env` 和 `config.yml` 提交到版本控制
   - 确保 `.gitignore` 正确配置

3. **网络安全**
   - 使用 HTTPS（通过 Cloudflare Tunnel 或反向代理）
   - 限制访问 IP（如果可能）
   - 定期更新依赖包

4. **数据安全**
   - 定期备份聊天记录和数据库
   - 不要在聊天中分享敏感信息
   - 定期清理旧日志

### API 密钥管理

- 使用环境变量或配置文件存储 API 密钥
- 不要在代码中硬编码密钥
- 定期轮换 API 密钥
- 使用最小权限原则

### 已知限制

1. **密码存储**：当前密码以明文形式存储在代码中，建议使用环境变量
2. **会话管理**：Cookie 有效期为 30 天，建议根据需求调整
3. **文件上传**：PDF 上传限制为 20MB，可根据需求调整
